## App Router 路由结构

### 页面路由

| 路由 | 文件 | 说明 |
|------|------|------|
| `/` | `src/app/page.tsx` | 首页 — 工具列表、搜索、分类筛选 |
| `/:category` | `src/app/[category]/page.tsx` | 分类页 — 该分类下的工具 |
| `/:category/:tool` | `src/app/[category]/[tool]/page.tsx` | 工具页 — 通用工具界面 |
| `/api-docs` | `src/app/api-docs/page.tsx` | API 文档页 |

### 路由渲染逻辑

1. **首页** (`page.tsx`)：客户端组件，fetch `/api/tools` 获取工具列表，支持搜索和分类筛选
2. **分类页** (`[category]/page.tsx`)：客户端组件，按 category 过滤工具
3. **工具页** (`[category]/[tool]/page.tsx`)：客户端组件，通过 `ToolPageContainer` 渲染通用工具界面
   - 特殊处理：`perler-beads` 使用独立的 `PerlerBeadsClient` 组件

## 工具注册系统

### 注册流程

```
1. 工具定义文件 (src/lib/tools/xxx.ts)
   ↓ 定义 Tool 对象 + register(tool)
2. 工具索引 (src/lib/tools/index.ts)
   ↓ import './xxx' (触发注册)
3. 注册表 (src/lib/registry.ts)
   ↓ tools Map 存储
4. API 路由 (src/app/api/xxx/route.ts)
   ↓ createToolRoute('xxx') 或自定义路由
5. 前端页面
   ↓ fetch /api/tools → /api/xxx
```

### 核心文件

**`src/lib/registry.ts`** — 工具注册表
```typescript
const tools = new Map<string, Tool>();

export function register(tool: Tool) { tools.set(tool.name, tool); }
export function getTool(name: string): Tool | undefined { return tools.get(name); }
export function getAllTools(): Tool[] { return Array.from(tools.values()); }
export function getToolsByCategory(category: string): Tool[] { ... }
```

**`src/types/tool.ts`** — 工具类型定义
```typescript
interface Tool {
  name: string;           // 工具唯一标识
  description: string;    // 工具描述
  category: 'pdf' | 'image' | 'video' | 'audio' | 'text' | 'dev' | 'convert' | 'file' | 'craft';
  inputSchema: z.ZodSchema;  // Zod 验证 schema
  execute: (input: any) => Promise<ToolResult>;  // 执行函数
}
```

### 工具开发五步法 (来自 CLAUDE.md)

1. **实现工具** — `src/lib/tools/<tool-name>.ts`
2. **创建 API 路由** — `src/app/api/<category>/<tool-name>/route.ts`
3. **注册工具** — `src/lib/tools/index.ts` 中添加 `import './<tool-name>'`
4. **添加 i18n** — 4 个语言文件中各添加 2 个 key
5. **编写测试** — `tests/tools/<tool-name>.test.ts`

## API 路由结构

### 通用模式

大部分 API 路由使用工厂函数 `createToolRoute()`：

```typescript
import { createToolRoute } from '@/lib/api-utils';
export const POST = createToolRoute('<tool-name>', { validate: true });
```

`createToolRoute` 支持三种模式：
1. **简单 JSON**：`parseJsonInput → tool.execute → JSON 响应`
2. **验证表单**：`parseInput → validateToolInput → tool.execute → buffer/JSON 响应`
3. **文件+字段**：`parseFormDataFile → fieldTransforms → tool.execute → buffer 响应`

### API 路由列表

| 路由路径 | 方法 | 工具 |
|----------|------|------|
| `/api/tools` | GET | 获取所有工具列表 |
| `/api/health` | GET | 健康检查 |

## i18n 国际化系统

### 实现方式

- **Provider**: `src/lib/i18n.tsx` — React Context
- **翻译文件**: `src/lib/locales/{zh,en,ja,ko}.json`
- **持久化**: `localStorage('furinakit-locale')`

### 使用方式

```typescript
const { t, locale, setLocale, tError } = useI18n();
t('tool.pdf-merge')        // 获取翻译
tError('INVALID_INPUT')    // 获取错误翻译
```

### 翻译 Key 约定

- 工具名：`tool.<name>` (如 `tool.pdf-merge`)
- 工具描述：`tool.<name>.desc`
- 导航分类：`nav.<category>` (如 `nav.pdf`)
- 错误消息：`error.<code>` (如 `error.INVALID_INPUT`)
