# FurinaKit 用户体验优化报告

> 审查日期：2026-05-31
> 视角：普通用户（小白 + 进阶用户）

---

## 一、首页（工具列表页）

### 1. 第一次打开，知道怎么用吗？

**评分：4/5**

首页布局清晰：Logo + 副标题 + 搜索框 + 分类标签 + 工具卡片网格。用户能立即理解这是一个工具集合站。

**问题：**
- 缺少「新手引导」或「推荐工具」区域。首次访问的用户面对 37 个工具卡片，不知道从哪个开始
- 「最近使用」区域在没有使用记录时不显示，这是正确的，但缺少「热门工具」或「入门推荐」作为替代
- metadata 声称「108 个实用工具」，实际只有 37 个，误导用户

**改进建议：**
- 添加「推荐工具」行（如 JSON 格式化、图片压缩、PDF 合并等高频工具）
- 修正 metadata 中的工具数量描述
- 考虑添加工具使用次数统计，展示热门排序

### 2. 搜索与筛选

**评分：4/5**

搜索框支持名称和描述的模糊匹配，分类标签支持点击筛选，交互流畅。

**问题：**
- 搜索框没有「清除」按钮（×），用户清空搜索需要手动选中删除
- 分类标签中「Audio」「Convert」点击后显示空列表（无对应工具），用户会困惑
- 缺少搜索结果计数（如「找到 5 个工具」）

**改进建议：**
- 在搜索框右侧添加清除按钮
- 为空分类标签添加工具数量角标，或灰化/隐藏空分类
- 在工具网格上方显示结果计数

### 3. 工具卡片

**评分：4/5**

卡片设计简洁，有图标、名称、描述、分类标签，hover 有阴影效果。

**问题：**
- 卡片无法通过键盘 Tab 导航聚焦（缺少 `focus-visible` 样式）
- 分类标签颜色方案虽然美观，但颜色含义不直观（用户需要猜测红色=PDF、蓝色=Image 的对应关系）

**改进建议：**
- 为卡片添加 `focus-visible` 样式，支持键盘导航
- 考虑在分类标签旁添加小图标（已在侧边栏实现，卡片上也可以复用）

## 二、通用工具页面框架

### 4. 工具页面布局

**评分：3.5/5**

工具页面使用 Card 组件包裹，有标题、描述、输入区、选项区、执行按钮、结果区，层次分明。

**问题：**
- **所有工具共用一个巨大的 `page.tsx` 文件（1574 行）**，通过 `switch(toolName)` 渲染不同选项。这意味着：
  - 每次工具切换都加载整个文件
  - 新增工具需要修改这个巨型文件
  - 选项渲染逻辑与页面逻辑耦合
- 工具标题显示的是 `tool.name`（英文 slug 如 `json-format`），而不是翻译后的名称。用户看到的卡片是「JSON 格式化」，进入页面后标题变成 `json-format`
- 缺少「返回首页」的明显按钮（只有面包屑导航）

**改进建议：**
- 将工具标题改为 `t('tool.${toolName}')` 显示翻译名称
- 将每个工具的选项渲染拆分为独立组件
- 在工具页面顶部添加明显的「← 返回」按钮

### 5. 操作步骤

**评分：4/5**

大多数工具的操作流程是：上传/输入 → 设置选项 → 点击执行 → 查看结果。步骤最少化。

**问题：**
- 对于 `uuid-gen`、`password-gen`、`timestamp` 等自动生成工具，用户仍然看到一个灰色的「执行」按钮，不知道这些工具是自动的
- `text-case` 和 `text-count` 也是自动提交的，但 UI 上没有明确提示
- Ctrl+Enter 快捷提交是好的，但没有在界面上提示

**改进建议：**
- 对自动提交工具，将按钮改为「已自动执行」或直接隐藏按钮
- 在输入区域下方添加提示文字：「修改选项后自动更新结果」或「按 Ctrl+Enter 执行」
- 考虑为所有工具添加「实时预览」模式（输入即输出）

### 6. 结果展示

**评分：4/5**

结果区域有「复制」和「下载」按钮，JSON 格式化有树形视图 + 原始视图切换，文本对比有 DiffViewer，这些都很优秀。

**问题：**
- 图片工具处理后自动下载文件，但结果区域没有显示处理后的图片预览（只有 `image-convert`、`image-compress` 等部分工具有预览）
- 文件类工具的结果是「结果为文件，点击下方下载」，但用户可能不知道处理是否成功
- 处理成功后没有「文件大小对比」信息（如压缩工具应该显示「原始: 2.3MB → 压缩后: 1.1MB，减少 52%」）

**改进建议：**
- 为所有图片工具统一添加处理前后对比预览
- 为文件类工具添加处理结果摘要（文件大小、处理时间等）
- 添加处理进度条（特别是大文件处理时）

## 四、全局 UX 问题

### 7. 国际化（i18n）问题

**评分：3/5**

支持中/英/日/韩四语言，覆盖全面。

**问题：**
- 拼豆工具中「空心圆」「高度」「格（宽按原图比例自动计算）」等标签硬编码为中文，未使用 i18n key
- `perler-client.tsx` 第 269 行的「自定义」未国际化
- 韩语翻译中缺少 `tool.perler-beads` 和 `tool.perler-beads.desc` 翻译键
- 部分术语解释（`term.*`）在韩语中缺失

**改进建议：**
- 扫描所有 `.tsx` 文件中的硬编码中文字符串，提取为 i18n key
- 补全韩语翻译中缺失的键
- 添加 i18n lint 规则，防止硬编码字符串

### 8. 侧边栏

**评分：3/5**

侧边栏有 Logo、分类导航、语言切换、版本号和工具计数。

**问题：**
- 「Audio」「Convert」「File」分类在侧边栏中显示，但点击后显示空页面（无对应工具）
- 侧边栏在移动端隐藏（`hidden lg:block`），但没有移动端替代导航（如底部导航栏或汉堡菜单）
- 语言切换器放在侧边栏底部，不够显眼

**改进建议：**
- 为空分类添加「即将推出」标签，或直接隐藏
- 添加移动端导航（底部 Tab 栏或顶部汉堡菜单）
- 将语言切换器移到更显眼的位置（如顶部导航栏）

### 9. 错误处理

**评分：3.5/5**

有统一的错误码翻译系统（`error.*`），使用 toast 显示错误。

**问题：**
- 错误信息过于技术化（如「PROCESS_FAILED」对普通用户无意义）
- 大文件上传时没有文件大小限制提示，可能直接报错
- 网络错误时没有重试按钮

**改进建议：**
- 将错误信息改为用户友好的描述（如「文件处理失败，请检查文件格式是否正确」）
- 在文件上传前检查大小限制，提前提示
- 添加错误时的「重试」按钮

### 10. 性能感知

**评分：3.5/5**

**问题：**
- 工具列表通过 API 加载，首次访问有明显延迟（显示 loading spinner）
- 大文件处理时没有进度反馈，用户不知道是卡了还是在处理
- 图片工具的预览通过 FileReader 读取为 base64，大图片可能导致页面卡顿

**改进建议：**
- 考虑将工具列表静态化（SSG），减少 API 调用
- 为文件处理添加进度条或骨架屏
- 对大图片预览进行压缩处理

## 五、高级用户功能

### 11. 快捷方式

**评分：3/5**

- ✅ Ctrl+Enter 快捷提交（已实现但未提示）
- ✅ 自动提交工具（uuid-gen、password-gen、timestamp、text-case、text-count）
- ❌ 缺少键盘快捷键导航（如 Tab 切换选项、Esc 清除结果）
- ❌ 缺少 URL 参数支持（如 `?text=hello&case=upper` 直接获取结果）
- ❌ 缺少 API 文档（开发者可能想直接调用 API）

**改进建议：**
- 在界面上显示快捷键提示
- 支持 URL 参数直接调用工具
- 添加 `/api-docs` 页面，暴露 REST API 文档

## 六、总结

### 总体评分：3.8/5

| 维度 | 评分 | 说明 |
|------|------|------|
| 首次使用引导 | 4/5 | 布局清晰，但缺少推荐和引导 |
| 操作效率 | 4/5 | 步骤精简，自动提交体验好 |
| 结果展示 | 3.5/5 | 文本工具优秀，文件工具有提升空间 |
| 错误处理 | 3.5/5 | 有统一框架，但信息不够友好 |
| 国际化 | 3/5 | 覆盖全面，但有遗漏和硬编码 |
| 移动端适配 | 2.5/5 | 响应式布局基本可用，但缺少移动端导航 |
| 高级功能 | 3/5 | 有快捷键，但缺少 URL 参数和 API 文档 |

### 优先改进建议（Top 10）

1. **修复工具标题显示**：工具页面标题从 `tool.name` 改为翻译名称
2. **修正 metadata 工具数量**：从「108」改为实际数量
3. **隐藏空分类**：Audio、Convert 等无工具的分类应隐藏或标记
4. **补全硬编码中文**：拼豆工具中的硬编码字符串提取为 i18n key
5. **添加文件大小对比**：压缩类工具显示处理前后大小
6. **添加移动端导航**：底部 Tab 栏或汉堡菜单
7. **统一图片工具预览**：所有图片工具添加处理前后对比
8. **添加快捷键提示**：在输入区域显示 Ctrl+Enter 提示
9. **改进错误信息**：技术错误码转为用户友好描述
10. **添加推荐工具区域**：首页展示高频工具推荐
